# 图例阅读方法

> **定位：** 道氏理论读图不是给曲线贴标签，而是从指数图上按顺序读出级别、方向、确认、等待与失效。

## 1. 本次范围
- 覆盖：指数图上的级别识别、方向判读、双指数确认、横盘等待、离区后的延续与失效。
- 不覆盖：K 线细部形态交易、个股短线执行、A 股固定互证组合设计。

## 2. 核心定义
- **图例阅读方法：** 把指数走势图从“涨跌线条”转成“趋势结构与确认关系”的固定阅读顺序。
- **核心问题：** 当前属于主要运动、次级运动还是日间波动；当前动作是否已被双指数确认；当前突破/回落是趋势延续，还是仍处在横盘或未确认状态。
- **项目内翻译：** 读图的目标不是给局部形态命名，而是给上游判断提供 `movement_level`、`trend_bias`、`confirmation_state`、`range_state`、`invalidation_if` 与等待原因的约束。

## 3. 结构单元

### 推荐阅读顺序
1. **先看级别。** 当前图上最重要的是主要运动，还是中级反应。
2. **再看方向。** 高点和低点是在顺向递进，还是结构被破坏。
3. **再看双指数关系。** 一个指数先动了，还是两个指数都动了。
4. **再看区间。** 当前是否处于窄幅横盘，边界在哪里。
5. **最后看离区后的反馈。** 突破/跌破后是否留在区间外，并得到后续延续。
6. **若做 A 股适配，再看互证对象是否合理。** 先确认比较的是相关主指数，而不是随手拼接的无关对象。

### 关键阅读原则
- 同样的上涨，处在熊市次级反弹里和牛市主升段里，意义不同。
- 同样的破位，若只有单一指数出现，判断应下调。
- 图例最重要的不是短时冲击，而是它是否改变更高层级结构。
- 横盘要配合后续突破与确认来看，不能把横盘本身当方向判断。
- 双重顶双重底更适合作为辅助警示，不应越权替代主趋势与指数确认。
- 量价活跃度只能辅助理解推进或衰竭质量，不能越过价格与确认链路单独裁决。

## 4. 图形理解

### 图片中必须识别的结构
- 当前波动属于主要运动、次级运动还是日间波动。
- 高点和低点是否仍沿原方向推进。
- 当前是否处于横盘区间，边界是否清晰。
- 突破/跌破后是否留在区间外，并得到后续延续。
- 两个关键指数是否共同支持当前判断。

### 常被忽略的隐含信息
- 强反弹若仍处在熊市次级运动里，其意义应低于牛市主升段中的同幅上涨。
- 单一指数先行越位，更接近候选信号，而不是确认判断。
- 横盘的价值在于“市场暂时平衡”，而不是“方向已经自动给出”。
- 若领先指数之后很快回落，落后指数也未跟上，原动作更应降级。

## 5. 触发与失效

### 触发条件
- 图上长周期方向相对清晰。
- 当前动作开始接近关键前高/前低或横盘边界。
- 两个关键指数中至少有一个开始出现值得等待确认的同向动作。

### 确认条件
- 不能跳过级别判断，直接从局部波动得出总趋势判断。
- 必须比较两个关键指数是否同向确认。
- 必须确认突破后的持续性，而不是只看盘中或单日越位。
- 若做 A 股适配，必须先说明互证对象为什么能代表同一趋势背景；若互证对象本身不稳，应先保留 `confirmation_state=unconfirmed`，并补 `wait_reason=invalid_pairing`。

### 失效条件
- 只凭单日长阳/长阴读图。
- 只凭单一指数新高/新低上调判断强度。
- 忽略次级运动，把中级反应误写成主趋势反转。
- 忽略横盘边界与离区后的后续反馈。
- 在 A 股里使用缺乏互证关系的对象后，仍保留原确认判断不降级。
- 明明互证对象本身不稳，却不补 `wait_reason=invalid_pairing`，仍保留强确认判断。

### 常见假信号
- 看到强反弹就忘了大级别仍在熊市。
- 看到单一指数领涨就提前宣布确认。
- 看到横盘时间久，就默认一定会出现大行情。
- 看到双顶双底外观，就跳过趋势与确认链路。
- 在 A 股里随意拿两个不具备互证关系的指数拼接成“确认”。

## 6. 与其他体系协同
- 与 [[威科夫操盘法]]：威科夫更细看区间内部事件，道氏理论先读大方向与双指数互证。
- 与 [[量价分析]]：量价可帮助理解突破质量，但先后顺序仍是“级别 → 方向 → 确认 → 质量”。
- 与 [[艾略特波浪理论]]：波浪更擅长细分内部结构，道氏理论更适合先做全局读图框架。

## 7. AI 可用映射
- 可沉淀为程序字段：`movement_level`、`trend_bias`、`confirmation_state`、`range_state`、`invalidation_if`、`false_signal_risk`。
- 可沉淀为 `skill_snapshot`：主要趋势推进 / 次级反应 / 横盘等待 / 单指数未确认 / 互证未稳。
- 可沉淀为 `decision_context`：当前是否可能把次级运动误判成反转；当前离区动作是否已经得到另一指数与后续延续支持；若 A 股互证对象本身不稳，应先保留 `confirmation_state=unconfirmed`，再补 `wait_reason=invalid_pairing`。
- 仅保留在解释层：误导性外观与领先动作为何不能直接上调确认的说明。

## 8. 暂未解决问题
- 图例页更偏阅读顺序与输入框架，不适合在本轮硬写大量数值阈值。
- A 股短线脉冲与情绪波动强，后续仍需补“收盘后结构 + 1–3 个交易阶段延续”的边界校准。
- 是否阻塞主线：**否**。

## 9. 后续优化提示
- 若未来补案例，应优先用本页阅读顺序去标注现有主线页，而不是单独扩大量图例故事页。

## 11. 上游消费提示
- 若上游输入主要来自图像或 OCR，先按“级别 → 方向 → 确认 → 区间 → 失效”顺序输出，不要先写判断。
- 若图上只有单一指数先动，优先保留 `confirmation_state=unconfirmed`，不要跳过互证链路。
- 若互证对象本身不稳或图上仍在横盘中，先保留 `confirmation_state=unconfirmed`，并补具体 `wait_reason`，不要把局部越位写成强确认。

## 12. 本轮收口结论
- 当前已完成的主线：把图例阅读固定为“级别 → 方向 → 双指数确认 → 横盘等待 → 失效下调”的顺序化框架。
- 允许暂停的未解问题：A 股互证对象细化、图例阈值量化。
