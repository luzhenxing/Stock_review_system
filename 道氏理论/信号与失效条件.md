# 信号与失效条件

> **定位：** 本页把道氏理论中真正可上调判断强度的信号、必须等待的确认、以及应当撤销判断的失效条件统一收口。

## 1. 本次范围
- 覆盖：三种运动、双指数确认、趋势判断、窄幅横盘、量价关系、双重顶底的主线收口。
- 不覆盖：个股筛选与执行策略细节。

## 2. 核心定义
- **触发条件：** 值得进入候选判断的初步动作。
- **确认条件：** 可以把候选上调为相对稳定判断的必要补充。
- **失效条件：** 后续价格行为否定原判断，使其必须撤销或降级。

## 3. 结构单元

### 1. 趋势延续信号
- 双指数继续同向推进。
- 高低点关系未被破坏。
- 次级运动结束后，原方向重新恢复。

### 2. 趋势反转候选
- 原趋势结构开始出现破坏迹象。
- 一个指数先行转弱/转强，但仍处在未确认状态。
- 横盘后的离区动作提示方向可能变化。

### 3. 横盘突破候选
- 价格离开 line 区间。
- 但若无确认与后续留区，仍只能是候选，而不是判断。

### 4. 辅助警示信号
- 量价关系异常。
- 双重顶/双重底外观。
- 单一指数背离。
- 消息冲击或短期异常波动。

> **说明：** 这些更适合作为风险提示或注意事项，不应替代双指数确认主链路。

## 4. 图形理解

### 图片中必须识别的结构
- 当前动作属于趋势延续、未确认转折，还是横盘等待。
- 一个指数先动，还是两个指数已经共同支持同一方向。
- 当前离区动作是否只是瞬时越位，还是已经留在区间外并获得后续延续。
- 原趋势高低点关系是否仍在，还是已经开始被结构性改写。

### 图片隐含表达
- 真正重要的不是“看起来很强/很弱”，而是这种强弱有没有被确认链路承接。
- 横盘后的离区动作若无后续留区，更接近假信号而不是方向判断。
- 单一指数背离、双顶双底或量价异常，更适合先视作警示，而不是直接替代主链路。

## 5. 触发与失效

### 必须满足
- 两种指数方向一致。
- 后续价格延续原判断，而不是迅速否定。
- 当前动作不只是日间波动或单次脉冲。
- 若做 A 股适配，互证对象必须能说明其处在同一趋势背景，而不是任意拼接两个指数。

### 加分条件
- 横盘离区后持续站稳区间外侧。
- 次级运动结束后，原趋势迅速恢复推进。
- 量价表现对突破方向形成辅助支持。
- 推进时更活跃、逆向反弹或回调时更迟钝，有助于提高原判断质量。

### 必须撤销或降级的情况
- 单一指数动作迟迟得不到另一指数确认。
- 突破/跌破后快速回到原区间。
- 原趋势高低点关系被结构性改写。
- 被视作次级运动的动作，实际演化为更高层级的趋势破坏。

### 更适合作为降级判断的情况
- 当前更像未确认转折，而非已确认反转。
- 当前更像熊市反弹 / 牛市回调，而非新主趋势起点。
- 当前更像横盘等待，而非已完成方向选择。
- 当前互证对象本身不稳，更适合先保留 `confirmation_state=unconfirmed`，再补 `wait_reason=invalid_pairing`，而不是继续上调方向判断。

## 6. 常见假信号
- 盘中快速拉升/跳水。
- 单一指数先行新高/新低。
- 缺乏后续延续的 line 突破。
- 只凭双重顶/双重底外观得出确定方向。
- 用消息或叙事替代价格与确认链路。
- 把熊市中的缩量反弹或牛市中的放量衰竭误写成主趋势已经反转。

## 7. 与其他体系协同
- 与 [[量价分析]]：量价异常可作为辅助警示，但要回落到“是否确认、是否失效”。
- 与 [[威科夫操盘法]]：威科夫里的假突破、测试失败等，可帮助细化道氏理论中的“未确认/失效”解释层。
- 与 [[艾略特波浪理论]]：浪形给出结构猜测，道氏理论负责用确认与失效约束猜测强度。

## 8. AI 可用映射
- 可沉淀为程序字段：`signal_type`、`confirmation_state`、`invalidation_if`、`false_signal_risk`。
- 可沉淀为 `skill_snapshot`：趋势延续 / 未确认转折 / 横盘等待 / 互证未稳 / 假信号风险上升。
- 可沉淀为 `decision_context`：还缺什么确认、哪里最容易失效；若 A 股互证对象本身不稳，应先保留 `confirmation_state=unconfirmed`，并补 `wait_reason=invalid_pairing`。
- 仅保留在辅助证据层：量价细节、情绪解读、双顶双底叙事。

## 9. 暂未解决问题
- 量价关系在道氏理论中的具体权重，后续需要与独立量价体系做边界切分。
- A 股中的多指数确认是否应加入宽基 + 行业/风格互证，仍待后续校准。
- 是否阻塞主线：**否**。

## 10. 后续优化提示
- 若未来程序化，可把“确认不足”与“失效已发生”明确区分，不要混成一个弱信号桶。
- 若未来进入 AI 联判，道氏理论最适合提供方向性约束与撤销条件，而不是直接替代执行层。

## 12. 上游消费提示
- 若当前只是候选动作，优先输出 `signal_type` 或 `confirmation_state=unconfirmed`，不要直接跳到确认判断。
- 若还缺另一指数承接或结构延续，先保留 `confirmation_state=unconfirmed`，必要时补具体 `wait_reason`。
- 若原判断已被后续动作破坏，应优先补 `invalidation_if` 或上调 `false_signal_risk`，不要继续沿用旧判断。
