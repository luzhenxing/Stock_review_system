# 谐波技能

> **定位：** 对应代码中的 `skills.harmonic`，用于描述当前是否存在可用的 XABCD 谐波候选、完成度、PRZ 与失效线索。
> 它保持为独立 skill，不并入波浪理论，也不直接输出最终操作指令，而是在 AI+skill 联判中提供反转区证据。
> 当前在分析链路中已不再只作为纯摘要文本，而是以 **候选形态 + completion + PRZ + invalid_reason** 的证据形式进入知识上下文与失效约束层。

---

## 当前代码输出字段

```python
skills.harmonic = {
    "enabled": True,
    "pattern": "Gartley/Bat/Butterfly/.../none",
    "direction": "bullish / bearish / neutral",
    "completion": 0.0,
    "prz": {"low": ..., "high": ...} | None,
    "fit_score": 0.0,
    "evidence": ["pattern", "invalid_reason"],
    "summary": "候选形态｜PRZ low-high",
}
```

---

## 触发与观察依据

谐波技能当前基于以下信息统一生成：

1. 最近摆点构成的 XABCD 候选结构。
2. Fib 比率是否接近目标谐波约束。
3. 价格是否进入或接近 PRZ。
4. 当前是否存在明显失效原因。

它回答的是：
- 有没有值得关注的候选形态？
- 当前是多头反转候选还是空头反转候选？
- PRZ 是否已经形成，结构完成度如何？

---

## 在 AI 结论中的角色

谐波技能主要负责回答 **“反转区是否足够精确”**：

- `pattern != none` 且 `prz` 明确时，AI 会把它视作结构共振证据。
- `completion` 与 `fit_score` 越高，越适合作为反转观察重点。
- 如果 `evidence` 中包含失效原因，则更偏向观察或否定，而不是直接参与追涨。
- 进入 PRZ 或命中 D 点，只代表价格进入候选完成区；若仍存在 `invalid_reason`，则程序与 AI 都应继续停留在候选/观察语义。
- 当前分析链路会把谐波失效线索并入 `decision_layer.invalidations`，但仍不允许谐波单独改写主动作。

---

## 与 feature_gate 的边界

- `skills.harmonic` 负责 **图形结构与 PRZ 描述**。
- `feature_gate` 负责 **量价/威科夫/风险侧辅助解释**。

因此：
- 谐波结构可以完成，但若 `feature_gate` 指向 `blocked_distribution` 或 `blocked_effort`，AI 仍应警惕假反转。
- `feature_gate` 通过，也不代表谐波结构一定有效；没有候选形态时不能硬套谐波。

---

## 与其他 skill 的协同关系

- 与 `skills.wave`：波浪看大结构阶段，谐波看价格转折区是否精准。
- 与 `skills.chan`：缠论可补充段落位置与中枢背景，帮助判断谐波处于段内还是段尾。
- 与 `skills.ruthless`：无情操盘手可为 PRZ 附近提供更实际的执行触发和止损语言。

---

## 当前使用方式

- 看板展示中呈现 `harmonic_completion`、`harmonic_prz` 与 `skill_snapshot.harmonic`。
- AI 分析时把谐波 skill 作为“结构共振证据”输入，不作为唯一结论源。
- 知识注册表与运行时评估会抽取 `summary / completion / fit_score / prz / invalid_reason`，区分候选完成、接近完成与失效线索。
- `decision_layer.invalidations` 会吸收谐波失效提示，但主动作仍由程序门控、无情、威科夫、量价等更高优先级约束共同决定。
- 未来若继续扩形态库，应优先扩 producer，而不是在 consumer 端各自补判。
