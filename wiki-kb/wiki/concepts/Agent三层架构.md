# Agent 三层架构

XInGPT 在 [[个人业务Agent升级指南]] 中提出的个人 Agent 系统架构，将任何知识工作拆解为三层。

## 第一层：知识库（Knowledge Base）

Agent 的"记忆系统"。包含：

- **历史数据库**：过往数据、事件复盘笔记
- **重要指标与信息源**：关注的媒体、数据指标、关键人物
- **个人经验库**：决策记录 + 对错复盘

> 文章案例：50 万条结构化数据，每天自动更新 200+ 条，替代 2 个全职研究员。

## 第二层：Skills（决策框架）

Agent 的"判断标准"。把隐性的专业知识**显性化、结构化**为 if-then 规则：

```
输入: 特定类型的数据
判断标准: 可量化的条件列表
触发条件: 满足 N 个以上条件 → 信号
输出: 评级 + 建议动作
```

> 与 [[个性化 Agent 上下文]] 的关系：Skills 定义"能做什么"，上下文定义"如何思考"。

## 第三层：CRON（自动化执行）

Agent 的"手脚"。定时触发、自动运行、推送结果。

- 信息收集自动化：定时抓取数据源
- 预处理自动化：格式化、分类、初步分析
- 推送自动化：在需要决策时呈现给用户

## 分层原则

诊断自己的工作时按三个问题分类：
1. 是否可以标准化？→ 可 Agent 化
2. 是否需要创造性思考？→ 不需要则可 Agent 化
3. 是否需要我的独特判断？→ 不需要则可 Agent 化

**重复性工作优先 Agent 化，判断性工作人机协作，执行性工作自动化。**
