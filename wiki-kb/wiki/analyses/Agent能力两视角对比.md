# Agent 能力两视角对比

两篇来自同一公众号（[[中国人本来就聪明]]）的 Agent 深度文章，但切入角度和落点完全不同。

## 各自回答的核心问题

| | [[Agent 的超级进化]] | [[个人业务Agent升级指南]] |
|------|------|------|
| 作者 | 未署名（公众号原创） | [[XinGPT]]（转载） |
| 核心问题 | **Agent 自身如何进化和运行？** | **Agent 能为用户创造什么价值？** |
| 视角 | 内向：Agent 系统的内部构造 | 外向：Agent 对业务的实际改造 |
| 类比 | 如何造一辆更好的车 | 用这辆车跑出了什么成绩 |

## 架构对比

| 维度 | 超级进化 | 升级指南 |
|------|----------|----------|
| 架构表达 | CLAUDE.md + Rules + Skills 两层 | 知识库 → Skills → CRON 三层 |
| 知识管理 | `/eat` 机制 + Git 管理 | 50 万条结构化数据 + 每日自动更新 |
| Skills 定位 | 可组合的能力积木，渐进式披露 | 判断标准的显性化，if-then 规则 |
| 自进化 | 正负反馈闭环写入文件系统 | 数据驱动迭代（A/B 测试标题公式等） |
| 运行方式 | 长程独立运行（Long Horizon） | CRON 定时触发 + 人机协作 |

## 共识点

两篇文章在以下观点上高度一致：

- **Skills 是核心**：把专业知识结构化是 Agent 价值的关键
- **Agent 化需要沉淀时间**：不是一次性搭建，而是持续迭代的复利过程
- **人的角色从执行者变为决策者**：Agent 做 80% 的体力活，人做 20% 的判断
- **上下文是护城河**：模型能力会趋同，个人经验知识库才是差异化来源

## 互补关系

| 超级进化的贡献 | 升级指南的贡献 |
|----------------|----------------|
| 提供了 Agent 系统**如何搭建**的工程方案 | 提供了 Agent 系统**用来做什么**的业务蓝图 |
| 长程运行的技术路径 | 从诊断到商业化的四步行动清单 |
| Skill 的技术定义（L1/L2/L3 加载） | Skill 的业务模板（四个投资决策实例） |
| 上下文架构设计原则 | 内容生产的人机协作分工 |

## 一句话总结

> **超级进化**告诉你 Agent 可以怎么造，**升级指南**告诉你造出来后能替你做什么。两篇合在一起，构成了从工程实现到业务价值的完整闭环。
