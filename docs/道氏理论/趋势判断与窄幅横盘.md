# 趋势判断与窄幅横盘

> **定位：** 本页处理趋势延续、反转预警与 line（窄幅横盘）的关系，解决“什么时候应继续顺势，什么时候应等待突破确认”。

## 1. 本次范围
- 覆盖：第 13 章《判断趋势》、第 14 章《窄幅横盘》。
- 不覆盖：短线盘中策略、个股形态交易细节。

## 2. 核心定义
- **趋势判断：** 通过高低点关系、双指数确认与后续延续，判断主要方向是否仍成立。
- **窄幅横盘 / line：** 指数在较窄区间内持续整理，代表方向暂缓而非方向自动成立。

## 3. 结构单元

### 趋势判断主链路
1. 先确认当前大方向。
2. 再看高低点是否继续沿该方向推进。
3. 若进入横盘，先保留等待状态，不急于预测方向。
4. 待横盘突破并得到确认后，再恢复方向性判断。

### 横盘的角色
- 横盘是趋势中的暂停区，也是后续突破/跌破的准备区。
- 横盘本身不等于顶部或底部。
- 横盘后的突破若无确认，仍可能是假动作。

## 4. 图形理解

### 图片中必须识别的结构
- 高点与低点是否仍在顺向递进。
- 区间边界是否被有效突破。
- 突破后是否留在区间之外，还是迅速回到原区间。

### 图片隐含表达
- line 的价值在于“市场暂时达成平衡”，而不是“方向已经告诉你”。
- 真正重要的是离开横盘后，市场有没有继续沿突破方向延续。
- 若突破后很快回区，原突破更应降级为假信号或未确认动作。

## 5. 触发与失效

### 触发条件
- 趋势方向已由更大结构与双指数确认。
- 横盘整理结束后，价格有效离开区间。

### 确认条件
- 突破/跌破不只是一时越位，而是得到后续延续。
- 相关指数对离开横盘的方向给出同向支持。
- 若做 A 股适配，互证对象仍需合理；若互证对象本身不稳，应先保留 `confirmation_state=unconfirmed`，并补 `wait_reason=invalid_pairing`。

### 失效条件
- 趋势高低点关系被破坏。
- 横盘突破后迅速回区，且缺乏后续延续。
- 单一指数离开横盘，但另一指数不确认。
- 量价一度支持突破叙事，但价格结构与后续延续并未跟上。

### 常见假信号
- 盘中短暂突破横盘边界。
- 只看单一指数越位。
- 把横盘时间长短自动解释成趋势必然爆发。
- 把缩量反弹、放量冲高或双重顶底外观直接上调为方向判断。

## 6. 与其他体系协同
- 与 [[量价分析]]：量价可辅助判断离开横盘的质量与真假，但最终仍要回到确认主链路。
- 与 [[威科夫操盘法]]：威科夫会进一步区分吸筹/派发区间，道氏理论先处理“整理 → 突破 → 确认”的宏观版本。
- 与 [[艾略特波浪理论]]：横盘可对应调整结构，但是否恢复原趋势，仍需道氏式确认。

## 7. AI 可用映射
- 可沉淀为程序字段：`trend_state`、`range_state`、`confirmation_state`、`wait_reason`、`false_signal_risk`。
- 可沉淀为 `skill_snapshot`：趋势延续 / 横盘等待 / 已确认离区 / 互证未稳。
- 可沉淀为 `decision_context`：当前应顺势还是保留等待状态；若 A 股互证对象本身不稳，应先保留 `confirmation_state=unconfirmed`，并补 `wait_reason=invalid_pairing`。
- 仅保留在解释层：市场暂时平衡、情绪犹豫等故事语言。

## 8. 暂未解决问题
- line 的时长、宽度在原书中偏经验定义，当前不宜硬编码绝对阈值。
- A 股中假突破更频繁，后续需要结合收盘与次日反馈补边界。
- 是否阻塞主线：**否**。

## 9. 后续优化提示
- 若未来进入看板或 AI 分析，可把“横盘中”显式标为等待状态，避免系统过度下方向判断。
- 若半程序化，优先识别“离开区间后是否留区并获得确认”。

## 11. 上游消费提示
- 若当前仍在横盘中，优先输出 `range_state=in_line` 或保留等待状态，不要抢写方向突破。
- 若离区后还缺双指数承接，先保留 `confirmation_state=unconfirmed`；若仍未真正离区或很快回区，再按情况补 `wait_reason=still_in_line` 或其他等待原因。
- 若突破后迅速回区，应优先上调 `false_signal_risk` 或补 `invalidation_if`，不要维持强趋势措辞。
