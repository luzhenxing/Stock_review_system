# 缠论技能字段与角色

> **定位：** 对应代码中的 `skills.chan`，用于给出最近段落推进方向、中枢数量与简化结构证据。
> 当前版本是面向联判的轻量缠论 skill，还不是完整缠论教学体系。

---

## 当前代码输出字段

```python
skills.chan = {
    "enabled": True,
    "state": "上段推进 / 下段延伸 / 盘整",
    "segment_bias": "bullish / bearish / neutral",
    "zs_count": 0,
    "evidence": ["recent_highs=...", "recent_lows=...", "zs=..."],
    "summary": "状态｜中枢 N 个",
}
```

---

## 触发与观察依据

当前 `skills.chan` 由 producer 基于最近 swing 高低点轻量构建：

1. 取最近若干摆点高低点。
2. 比较最新收盘与最近顶部/底部的接近程度。
3. 估算最近中枢数量 `zs_count`。
4. 输出当前更像上段推进、下段延伸还是盘整。

它回答的是：
- 最近段落偏向上行、下行还是盘整？
- 当前走势更像段内推进还是区间纠缠？
- 最近结构中枢数量大致如何？

---

## 在 AI 结论中的角色

缠论 skill 当前主要负责回答 **“最近段落处于什么演化状态”**：

- `state=上段推进` 时，更接近顺势段内延续证据。
- `state=下段延伸` 时，更接近弱势延续或风险提示。
- `state=盘整` 且 `zs_count` 较高时，AI 更应避免把走势描述成单边趋势。

---

## 与 feature_gate 的边界

- `skills.chan` 负责 **段落/中枢语义**。
- `feature_gate` 负责 **风险观察与辅助通过标签**。

因此：
- 缠论段落可以偏多，但若 `feature_gate` 显示派发/努力无结果，AI 不应只因段落上行就给强结论。
- `feature_gate` 看多，也不能替代中枢与段落本身。

---

## 与其他 skill 的协同关系

- 与 `skills.wave`：缠论补最近段落，波浪补更高层级结构位置。
- 与 `skills.harmonic`：缠论描述结构推进/盘整背景，谐波描述是否进入精确反转区。
- 与 `skills.ruthless`：缠论提供背景，无情操盘手提供执行信号和纪律。

---

## 当前限制

当前版本仍是轻量 schema：

- 还没有完整展开笔、线段、中枢分解过程。
- 还没有独立买卖点体系页与图解页。
- 更适合作为 AI+skill 联判中的结构补充层，而不是单独交易系统输出。
- 当上游文字出现吸货、做底、洗筹、负成本筹码、洗老鼠仓等故事语汇时，不能把它们直接当成 `skills.chan` 的可执行结论；必须同步回落到压力位、假突破/假破位、平台破坏、套牢区脱离速度与对应级别买卖点。
- 如果这些结构痕迹和级别落点说不清，相关表述只能留在解释层或背景层，不能冒充当前 skill 已稳定输出的事实。

后续若扩展，应优先保持 `skills.chan` 字段向后清晰，而不是在多个 consumer 端各自发明新口径。

---

## 终局收束后的字段边界

### 当前可稳定理解为 skill 事实的部分
- `state`：最近结构更像推进、延伸还是盘整。
- `segment_bias`：最近段落方向倾向的轻量摘要。
- `zs_count`：最近可简化表达的中枢数量信息。
- `evidence`：支撑轻量结论的结构证据摘要。
- `summary`：供上游快速消费的一句话结构概括。

### 当前不应冒充为 skill 事实的部分
- 完整分型—笔—线段—中枢推导链。
- 完整三类买卖点体系与跨级别传导。
- 只靠故事命名的做顶、吸货、围猎、洗筹、负成本筹码判断。
- 不能明确同级别结构痕迹、确认链与失效位的强操作结论。

### 后续扩展优先级
1. 先补结构确认与结构破坏语义。
2. 再补同级别失效约束与降级条件。
3. 最后才评估是否值得补更细的中枢状态或买卖点标签。

### 与知识库的分工
- 教材页级摘录负责证据回溯。
- 长期知识页负责定义、边界、顺序链与误判拦截。
- `skills.chan` 只消费其中已经稳定、低歧义、可重复表达的后段结构事实。
